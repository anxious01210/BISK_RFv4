One small note

Your debit transaction is being stored with:

tx_type = Debit
amount_iqd = 1000

That is fine if you want debit amounts stored as positive values and rely on tx_type + before/after balances to interpret direction.

But earlier your model design text suggested signed amounts for transactions, where a debit would be negative. Since the current behavior is already working, I would not change it now in the middle of testing. Just be aware that later you should pick one convention and keep it consistent everywhere.

About your idea of showing a small number badge

Yes — I think that is a good idea.

Best meaning for the counters

You could show:

confirm count
refund count

For wallet records, those counts can come from the ledger/history idea:

original debit
refund transaction
later re-debit if reconfirmed

For example:

✔ Confirmed (2)
Refunded (1)
