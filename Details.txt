OS related details:
2) “ffprobe not found in PATH” — what to do (Linux + Windows)

Your “Test RTSP” action calls ffprobe. It must be installed and visible on PATH for the Python process that runs Django.
Ubuntu 24.04

sudo apt update
sudo apt install ffmpeg
ffprobe -version   # should print a version
which ffprobe      # e.g., /usr/bin/ffprobe

Restart your Django server (or PyCharm run config) so it picks up PATH changes.
Windows 11

Choose one:

    winget (easy):

winget install --id Gyan.FFmpeg -e

Chocolatey:

    choco install ffmpeg

Ensure ffprobe.exe is on PATH (e.g., C:\ffmpeg\bin\). Close and reopen PyCharm/terminal so the env refreshes. Test:

ffprobe -version

Optional: allow an explicit path in settings

If you’d rather not rely on PATH (especially for cron/services), add in settings.py:

FFPROBE_PATH = "/usr/bin/ffprobe"  # or r"C:\ffmpeg\bin\ffprobe.exe" on Windows
